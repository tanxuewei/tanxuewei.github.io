读<javascript高级程序设计>笔记

2016-05-09 | Shirley Tan |

第一章

(1）p1

js诞生1995年。

以前由服务器来做输入验证操作，后来netscape公司希望js来做，因为那是拨号上网得年代，
通信很慢，所以格式不正确又发给服务端，服务端再反馈回来，会很耗时间，
验证放在客户端就完成就很好。

js又简单又复杂，简单是因为容易学，复杂是因为真正掌握需要数年时间。

要全面理解和掌握js，就要弄清它的本质，历史和局限性。

(2) p2

javascript的含义要比ecmascript范畴多得多，
因为完整得javascript实现包括三个部分，
ECMAScript+dom（文档对象模型）+bom（浏览器对象模型）

（4）p4
08年ie,firefox,safari,chrome,opera全部做到了与ecma-262兼容，
ie8是第一个实现ecma-262第五版本（es5）的浏览器。

（5）p5
dom是针对xml但经过扩展用于html得一种api接口。dom将页面映射为多层节点结构。
html页面就是由不同类型的节点构成的，每个节点又带着不同类型数据。
借助dom提供的接口，就可以轻松删除，添加，替换或者修改任何节点。

（6）p6
dom标准是负责制定web通信标准的w3c（world wide web consortium）组织规定的，
不是ecma组织制定。

dom并非仅针对js的，很多别得语言也实现了dom….啥语言啊？

（7）p7
SVG，MathML也都实现了dom

（8）p8
BOM是browser object model，浏览器对象模型。
bom以前没有标准，后来在h5里头得到了解决，h5把许多bom都写入正式规范。
bom范畴包括navigator对象，location对象，screen对象，cookies等
以及ajax得XMLHttpRequest。

bom是提供与浏览器交互得方法和接口。
dom是提供访问和操作网页内容得方法和接口。

第二章

（1）p10

defer表示脚本可以延迟到文档完全解析和显示后再执行，也是只对外部脚本有效。(会先于DOMContentLoaded事件，13章，p391)
script得defer属性是立即下载js，但延迟执行js。

async表示立即下载脚本，但不妨碍页面其他操作，只对外部脚本有效。
async 一定会在load前执行，但是可能会在DOMContentLoaded前后执行.

（2）p16
外部js得优点，
可缓存，A页面和B页面都引用了某个js，
则A页面下载js后，B页面可以直接使用缓存里得这个js。

而且可维护性也要好，一次修改，多处运行。
